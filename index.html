Introduction

Qu'est-ce que jQuery ?

jQuery = bibliothèque JavaScript
John Resig - 2006
Animations, manipulation du CSS, gestion d'évènements
Bibliothèque front-end la plus utilisée au monde

jQuery est une bibliothèque JavaScript libre et multiplateforme créée par John Resig en 2006.
Cette bibliothèque contient de nombreuses fonctionnalités, notamment des animations, la manipulation du CSS, la gestion des évènements, etc.
jQuery est à l'heure actuelle la bibliothèque front-end la plus utilisée au monde (plus de la moitié des sites Web intègrent jQuery).

À quoi ça sert ?

Simplifier la manipulation du DOM HTML
Gestion d’événements
Création d’animations CSS
Blocs de code JavaScript prêts à l'emploi
Gain de temps de développement
Compatibilité navigateurs

jQuery a été créé pour simplifier la manipulation du DOM HTML, la gestion d’événements et la création d’animations CSS.
La librairie consiste en un ensemble de blocs de codes JavaScript préconçus qui sont enfermés dans des méthodes.
Il suffit d’appeler ces méthodes pour exécuter le code qu’elles contiennent.
jQuery permet donc un gain de temps de développement et d’assurer une compatibilité de notre code avec les différents navigateurs.

Javascript dans tout ça ?

Pas une alternative au JavaScript

L'utilisation de jQuery est devenue moins pertinente depuis l'arrivée d'Angular et React.

Prérequis

Télécharger et inclure jQuery

Fichier compressé :
version de production du site web
Fichier non compressé :
version de développement du site web.

Pour pouvoir utiliser la librairie jQuery, il va falloir inclure un fichier jQuery dans la page HTML.
Il existe deux types de fichiers: un fichier compressé et un fichier non compressé.
La version compressée a pour but de réduire le poids total de la librairie afin d’améliorer les performances du site web. C’est la version utilisée sur un site en production.
La version non compressée est une version lisible de la librairie (commentaires, indentation, etc.).
A utiliser lors du développement du site web.

<head>
<script src="jquery-3.6.0.min.js"></script>
</head>

Inclure jQuery via un CDN

Content Delivery Network
2 avantages :
- pas de stockage de fichers
- optimisation des performances via la mise en cache du fichier

Une autre façon d’inclure jQuery est depuis un CDN (Content Delivery Network).
Utiliser un CDN plutôt que de télécharger et d’héberger les fichiers possèdent deux avantages notables : ne pas avoir à stocker les fichiers et optimiser les performances du site web.
Si on inclut jQuery via un CDN, les fichiers jQuery proviennent d’une URL commune.
Si un visiteur a précedemment visité un autre site utilisant le même CDN pour inclure le même fichier, alors le fichier est probablement en cache dans le navigateur.
Il n'y a donc pas besoin de télécharger le fichier à nouveau, notre page va s’afficher plus vite.

<head>
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.6.0/jquery.min.js"></script>
</head>

Fonctionnement

La syntaxe

$( "section" ) .hide()
Alias jQuery() ou $()
Ciblage d'éléments via les sélecteurs css
Appeler la méthode à appliquer à l'élément

Afin de pouvoir accéder aux différentes propriétés et méthodes de la librairie, on utilise les alias jQuery() ou $().
Ensuite, pour cibler un élément HTML, il suffit de mentionner le sélecteur CSS associé à cet élément dans notre méthode $().
Enfin, on appelle la méthode à appliquer à l'élément ciblé.

Les sélecteurs

Sélecteur d'élement : $("p")
Sélecteur de classe : $(".test")
Sélecteur d'ID : $("#test")

Afin de cibler l'elément auquel on veut appliquer un action, on utilise les sélecteurs CSS.
Parmis les sélecteurs, on trouve :
- Le sélecteur d'élement
- Le sélecteur de classe
- Le sélecteur d'ID

Je suis un paragraphe dans la div .test

Je suis un paragraphe dans la div #test

$(".test").hide()
$("#test").hide()
$("p").hide()
$("#test > p").hide()

Les évènements

Série de méthodes pour la gestion d'évènements
.click()
.mouseenter()
.mouseleave()

Avec jQuery, la gestion d'évènements est simplifiée grâce à une série de méthodes.
Généralement, ces méthodes portent le nom de l'évènement auquel elles réagissent.
Par exemple : click(), mouseenter(), mouseleave(), ... Similaire au addEventListener() en JavaScript Vanilla.
